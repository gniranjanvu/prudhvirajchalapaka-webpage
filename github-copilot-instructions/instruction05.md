## 11.3 Logout Flow

```yaml
Logout Process:

1. User clicks "Logout" in sidebar
   â†“
2. Confirmation modal appears
   â†“
3. If confirmed:
   - Clear session from Supabase Auth
   - Clear httpOnly cookie
   - Clear any local storage data
   - Redirect to /admin (login page)
   â†“
4. Show success toast: "You have been logged out successfully"

Auto-Logout Triggers:
  - Session expires (after 24 hours by default)
  - Session revoked from another device
  - User manually logs out
  - Security: Too many failed requests (potential attack)

After Logout:
  - All admin routes redirect to /admin login
  - User must request new OTP to login
  - Previous OTPs are invalidated
```
